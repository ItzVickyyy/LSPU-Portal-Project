About the University

A Legacy of Excellence in Laguna

Founded in 1952 in Siniloan, Laguna as a vocational and technical school, Laguna State Polytechnic University has grown into the province's leading state university — the only institutionally-accredited state university in the CALABARZON region, as certified by AACCUP.

With four campuses located in Santa Cruz, Siniloan, San Pablo City, and Los Baños, LSPU serves the CALABARZON industrial corridor and offers over 16 multidisciplinary undergraduate programs through 8 academic colleges.

LSPU holds ISO 9001:2015 certification and is recognized in international rankings including the Times Higher Education Impact Rankings and the WURI Global Top 500 Innovative Universities.

Our Campuses

Serving Laguna Province

LSPU has four main campuses and multiple satellite campuses strategically located across the province, serving students from all municipalities of Laguna.

Santa Cruz Campus
Santa Cruz, Laguna (Main)
The main campus and administrative center of LSPU. Houses the majority of colleges including Engineering, CTE, CBAA, and CAS. Home to the university's central administration and flagship programs.
Siniloan Campus
Siniloan, Laguna (Host)
The original host campus where LSPU was founded in 1952. Offers programs in Education, Business, Computer Studies, and Agriculture serving northern Laguna communities.
San Pablo City Campus
San Pablo City, Laguna
Serving the largest city in Laguna. Offers Computer Studies, Engineering, Business, Teacher Education, and Criminal Justice programs. Home to a DOST Advanced Manufacturing Center.
Los Baños Campus
Los Baños, Laguna
Located in the Special Science and Nature City of the Philippines. Formerly Los Baños School of Fisheries — now offering Fisheries, Computer Studies, Tourism, and Education programs.

From the Developer

Developer's Note

A quick look at how this project came to be — and everything it became along the way.

Where It Started

This project began as a final requirement for Advance Database System. The task was clear: design a normalized database and build web pages that perform basic CRUD operations connected to it. That scope alone took roughly two weeks of near-sleepless nights to finish properly. Rather than stopping at the minimum, I pushed to build something that felt complete — a real portal with authentication, email verification, role-based access control, an admin dashboard, and a polished interface throughout.

Three Subjects, One Project

Midway through development, three other subjects — Web Systems and Technologies, Integrative Programming Technologies 1, and Multimedia Systems — announced they would be merging their individual final project requirements into a single unified submission. The intent was to reduce student workload by consolidating three separate deliverables into one. Since my system already covered the scope of all three naturally, I carried the same project forward as my submission for the merged requirement as well.

The Requirement That Changed

Shortly after, the Advance Database System professor revised the deliverable. The final submission for that subject would only need to be the database schema and an Entity Relationship Diagram — not a working application. So the system I had been building no longer counted toward that subject at all. What started as an ADS project ultimately ended up submitted purely for the three merged subjects.

The Flask Complication

Then the formal instructions for the merged final project arrived — after the system was essentially finished. The instructions specified that the project must use Flask as the web framework, with a PC or laptop acting as the host server and a mobile phone on the same network as the client device.

The server-host and mobile-client requirements weren't a problem. The system was already designed to run locally on XAMPP, and the front-end was fully responsive from day one. That part was covered.

The Flask requirement, however, was a real conflict. The entire back-end had already been written in PHP. PHP and Flask are not interchangeable — they are fundamentally different technology stacks, and migrating from one to the other would mean rewriting the entire server-side codebase from scratch. After two to three weeks of development with the system already functioning well, that wasn't something I was willing to redo. The project already achieves every practical goal the instruction was aiming for: a local server hosting a web application accessed from a mobile device over a shared network. The framework used to serve it doesn't change what the system is or what it demonstrates.

As a small acknowledgment of the Flask requirement, the Status Checker module was built as a standalone Python/Flask application that connects to the same database and allows applicants to look up their application status without logging in. It runs alongside the main portal.

One More Requirement

And then came one final addition. Deployment became a requirement under Multimedia Systems — which is why this site exists online in the first place. What started as a local academic project is now live on the web.

Project Overview

About This Project

A full-stack student information portal built for Laguna State Polytechnic University — San Pablo City Campus.

Landing Page

A public-facing university landing page with university information, college listings, campus locations, and application entry points. Fully responsive across all devices.

Student Portal

A secure login portal for enrolled students. Students can view their profile, track their application progress, and access university services after authentication.

Application Page

A multi-step online application form for prospective students. Includes personal information, program selection, document submission, and account registration — all in one seamless flow.

Status Checker

A standalone Flask application that lets applicants check their admission status without needing to log in — just enter your reference number and see where you stand.

Registrar Panel

A dedicated dashboard for registrar staff to review incoming applications, update applicant statuses, manage records, and communicate decisions through the system.

Admin Panel

A full administrative dashboard with user management, system monitoring, and oversight tools. Admins can view activity across the portal and manage all user accounts.

Super Admin

A privileged role reserved for system-level management: creating new management accounts, assigning roles, and enabling or disabling access across the platform.

Tech Stack

Built with PHP and MySQL on XAMPP for the main portal, with a Python / Flask microservice for the Status Checker. The front-end uses plain HTML, CSS, and JavaScript — no frameworks.
